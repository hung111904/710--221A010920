Câu 1: Sự khác biệt giữa position: absolute và position: relative trong CSS

- position: relative: giữ phần tử trong luồng bình thường nhưng cho phép dịch chuyển nhẹ so với vị trí gốc.
- position: absolute: loại bỏ phần tử khỏi luồng và định vị nó theo phần tử cha gần nhất có thuộc tính position (không phải static).

Ví dụ với relative:


<div class="card">
  <img src="product.jpg" alt="Sản Phẩm">
  <p class="label">Sản Phẩm</p>
</div>

.card { position: relative; }
.label { position: relative; top: 10px; left: 5px; }

Ví dụ với absolute:


<div class="container">
  <img src="banner.jpg" alt="Banner">
  <button class="btn">Mua ngay</button>
</div>

.container { position: relative; }
.btn { position: absolute; bottom: 20px; right: 20px; }

Khi deploy website lên hosting: relative giúp layout ổn định hơn, absolute cần kiểm tra bằng media queries để tránh lỗi trên mobile.

Câu 2: Quy trình đăng ký domain và hosting

- Đăng ký domain qua nhà cung cấp (Namecheap, GoDaddy...).
- Chọn hosting (GitHub Pages, Netlify cho web tĩnh).
- Cấu hình DNS: ánh xạ domain → IP hosting.
- Công cụ hỗ trợ: GitHub Pages, Netlify.
- Rủi ro bảo mật: nếu không bật HTTPS hoặc cấu hình DNS sai → dễ bị tấn công giả mạo.

Câu 3: So sánh let, const và var về scope và hoisting

- var: phạm vi global hoặc function, hoisting với giá trị mặc định undefined.
- let: block scope, hoisting nhưng nằm trong Temporal Dead Zone.
- const: block scope, hoisting nhưng nằm trong Temporal Dead Zone, không gán lại.

Ví dụ lỗi thường gặp:


console.log(a); // undefined (do hoisting var)
var a = 5;

console.log(b); // ReferenceError (Temporal Dead Zone)
let b = 10;

console.log(c); // ReferenceError (Temporal Dead Zone)
const c = 15;

Với var, biến được hoisting nên có thể gọi trước khi khai báo, nhưng giá trị là undefined.
Với let và const, biến cũng được hoisting nhưng chưa thể truy cập trước khi khai báo → gây lỗi ReferenceError.

Cách debug bằng Chrome DevTools:
- Mở DevTools: Nhấn F12 hoặc Ctrl+Shift+I trong Chrome.
- Console tab: Chạy code để xem lỗi trực tiếp (ReferenceError, undefined).
- Sources tab: Đặt breakpoint tại dòng khai báo biến để quan sát giá trị trước và sau khi khởi tạo.
- Scope panel: Kiểm tra biến trong block scope, function scope, và global scope.
- Watch expressions: Thêm biến vào watch để theo dõi giá trị khi chạy từng bước.

Câu 4: Khái niệm Event Delegation trong JavaScript

Event Delegation là kỹ thuật trong JavaScript cho phép gắn một listener sự kiện vào phần tử cha, thay vì gắn trực tiếp vào từng phần tử con.

Khi sự kiện xảy ra, nó sẽ “bong bóng” (bubble) lên cha, và bạn có thể kiểm tra event.target để biết phần tử nào được click.

Điều này đặc biệt hữu ích khi làm việc với danh sách động (dynamic list), vì không cần gắn lại sự kiện mỗi khi thêm phần tử mới.

Ví dụ ứng dụng với danh sách động:


<ul id="product-list">
  <li>Sản phẩm A</li>
  <li>Sản phẩm B</li>
</ul>

const list = document.getElementById('product-list');
list.addEventListener('click', function(event) {
  if (event.target.tagName === 'LI') {
    alert('Bạn đã chọn: ' + event.target.textContent);
  }
});

// Thêm phần tử mới động
const newItem = document.createElement('li');
newItem.textContent = 'Sản phẩm C';
list.appendChild(newItem);

Ở đây, không cần gắn sự kiện cho Sản phẩm C vì listener ở cha (<ul>) sẽ tự xử lý.

Lợi ích:
- Giảm số lượng listener, tăng hiệu suất.
- Tương thích responsive khi thêm phần tử mới.
- Tối ưu UI/UX với CSS.
- Dễ bảo trì, code gọn gàng.
